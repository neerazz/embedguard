# Evidence that RAG poisoning / adversarial embeddings / indirect prompt injection are live production threats (2024-2026)

**Verified:** 2026-07-09 using live `curl`/PDF extraction against the URLs below.  
**Use case:** supporting EmbedGuard / OASec 2026 claims that RAG security is a real production problem, not a hypothetical.

## B. Industry threat frameworks that formally name the threat class

|---|---|---|---|---|

| OWASP LLM04:2025 Data and Model Poisoning | OWASP states data poisoning occurs when “pre-training, fine-tuning, or embedding data is manipulated” to introduce vulnerabilities, backdoors, or biases. | https://genai.owasp.org/llmrisk/llm042025-data-and-model-poisoning/ | OWASP Top 10 for LLM Applications 2025 | This directly names manipulated embedding data as an integrity risk, matching EmbedGuard’s adversarial embedding threat model. |
| OWASP LLM08:2025 Vector and Embedding Weaknesses | OWASP states vector/embedding weaknesses in RAG systems can be exploited to “inject harmful content, manipulate model outputs, or access sensitive information,” and lists data poisoning and embedding inversion as common risks. | https://genai.owasp.org/llmrisk/llm082025-vector-and-embedding-weaknesses/ | OWASP Top 10 for LLM Applications 2025 | This is the clearest industry taxonomy slot for an embedding/RAG security framework. |

| MITRE ATLAS AML.T0066 Retrieval Content Crafting | MITRE ATLAS describes adversaries writing content designed to be retrieved by user queries and influence the system/user; it can be combined with prompt injection or stand alone in documents/email. | Official ATLAS data: https://raw.githubusercontent.com/mitre-atlas/atlas-data/main/dist/ATLAS.yaml | Technique created 2025-03-12 | It formalizes malicious retrieved content as a named adversary technique, validating the paper’s focus. |
| MITRE ATLAS AML.T0070 RAG Poisoning | MITRE ATLAS describes adversaries injecting malicious content into data indexed by a RAG system to contaminate future threads through RAG search results. | Official ATLAS data: https://raw.githubusercontent.com/mitre-atlas/atlas-data/main/dist/ATLAS.yaml | Technique created 2025-03-12 | This is an exact technique-name match for the paper’s threat class and can anchor the motivation section. |

## D. Academic measurement studies showing attack feasibility at scale

| Study | What it demonstrated | Source URL | Date | How this motivates a RAG-security paper |
|---|---|---|---|---|
| PoisonedRAG: Knowledge Corruption Attacks to Retrieval-Augmented Generation of Large Language Models | The paper reports that injecting five malicious texts per target question into a knowledge database with millions of texts achieved a 90% attack success rate, and that evaluated defenses were insufficient; arXiv notes it is to appear at USENIX Security 2025. | https://arxiv.org/abs/2402.07867 | arXiv 2024-02-12; v3 2024-08-13; USENIX Security 2025 | It provides the strongest quantitative “scale” citation: millions of corpus texts, few poison documents, high ASR. |
| ConfusedPilot: Confused Deputy Risks in RAG-based LLMs | Demonstrated malicious enterprise documents changing Microsoft 365 Copilot behavior, causing incorrect answers, suppressing correct sources, leaking secrets through retrieval/cache behavior, and enabling misinformation propagation in a shared workspace. | https://arxiv.org/abs/2408.04870 | arXiv 2024-08-09; v5 2024-10-23 | It links RAG poisoning to everyday enterprise workflows and shows why defenses must reason about sharing, permissions, cache, and document provenance. |
| Phantom: General Backdoor Attacks on Retrieval Augmented Language Generation | Demonstrated a single poisoned document causing RAG integrity violations across multiple open-source LLMs, transfer to GPT-3.5/GPT-4, adversarial tool invocation 64% of the time, and black-box attack against NVIDIA ChatRTX with 48% success for passage exfiltration. | https://arxiv.org/abs/2405.20485 | arXiv 2024-05-30 | It supports the claim that one poisoned document can be enough and that production RAG systems can be vulnerable without white-box access. |
| Practical Poisoning Attacks against Retrieval-Augmented Generation / CorruptRAG | Proposes a practical single-poisoned-text-per-query RAG poisoning attack; experiments on three large-scale benchmark datasets and multiple LLMs show higher attack success than baselines and robustness against four defenses. | https://arxiv.org/abs/2504.03957 | arXiv 2025-04-04 | It shows the field is moving from multi-document poisoning toward more stealthy single-poison attacks, motivating stronger cross-layer detection. |
